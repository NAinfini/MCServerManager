# 应用日志工作区尺寸设计

## 目标

修正应用日志页在宽屏窗口中的空间分配：日志列表与详情面板应填满页面剩余高度，列表获得更多横向空间，同时保持窄屏布局可用。

## 布局规则

- 日志页使用完整可用高度，页头和筛选栏保持内容高度，工作区占据其余空间。
- 宽度大于 `1100px` 时，日志列表与详情面板按约 `60:40` 分配宽度；两栏均允许内容安全收缩。
- 日志列表取消 `620px` 固定最大高度，改为填满工作区并在内部滚动。
- 详情面板与工作区同高；长详情内容在面板内部滚动，不扩大页面。
- 宽度不大于 `1100px` 时恢复上下排列：日志列表保留受控高度，详情面板按内容高度展示，页面负责纵向滚动。

## 范围

仅修改应用日志页相关 CSS 和样式回归测试。不调整字体、按钮尺寸、日志分组逻辑、数据查询或其他页面布局。

## 验证

- 新增样式测试，覆盖完整高度链、`60:40` 宽屏比例、列表内部滚动和窄屏回退。
- 运行日志组件测试、完整 Vitest 测试和生产构建。
- 检查改动无横向溢出，不改变键盘焦点与现有交互。
