# 主内容区创建服务器与 BBSMC 接入设计

## 目标

将“创建服务器”从覆盖式模态框迁移到应用主内容区，同时把 BBSMC 加入创建向导的整合包来源。整个流程继续遵循现有六步向导：选择来源、兼容性检查、Java、服务器配置、EULA 检查、安装并启动。

成功标准：

- 点击“创建服务器”后，侧栏、窗口标题栏、顶部运行状态和底部状态栏保持可见，服务器列表由创建页面替换。
- 创建页面不再使用遮罩层、Portal、焦点陷阱或对话框语义。
- 未开始安装的草稿在离开前要求用户确认；安装任务开始后允许离开并继续后台执行。
- 来源下拉菜单同时提供 Modrinth 和 BBSMC。
- BBSMC 公共直链版本可以进入服务器创建规划；外部网盘版本不可自动安装。
- 无法确认是专用服务端包的 BBSMC 版本必须由用户确认风险后才能继续。

## 范围

本次修改包含：

- `AppShell` 中创建页面的打开、离开和返回状态。
- 创建页面头部、步骤指示器与向导内容的内联布局。
- 创建向导向外发布安装是否已经开始的状态。
- BBSMC 搜索、详情、版本选择和创建规划。
- 与上述行为直接相关的中英文文本、样式和测试。

不包含：

- 引入 React Router 或新的导航框架。
- 持久化尚未开始安装的向导草稿。
- 自动下载百度网盘、夸克网盘或其他外部网盘文件。
- 为 CurseForge 增加无凭据发现能力。
- 改变 Java、EULA 或服务器配置步骤的业务规则。

## 架构选择

采用 `AppShell` 内部页面状态，不引入路由依赖。创建流程属于服务器管理主页面的一个子视图，因此由 `AppShell` 在“服务器概览”“服务器详情”“创建服务器”之间切换最直接，也与当前应用结构一致。

不会通过修改模态框 CSS 来伪装成页面。创建流程将从 Radix Dialog 中移出，以消除不正确的对话框语义、遮罩层和焦点限制。

## 主内容区状态与导航

`AppShell` 保留现有服务器页面状态，并增加创建页面状态。打开创建页面时：

1. 当前主内容切换为创建页面。
2. 创建页面顶部渲染标题、描述、六步进度和返回控件。
3. `CreateServerWizard` 在页面主体中渲染，继续拥有具体步骤状态。
4. 拖入文件通过现有导入审查后直接打开创建页面，并把文件路径传入向导。

离开创建页面统一经过一个请求离开函数：

- 安装任务尚未创建：打开“放弃创建”确认对话框。确认后卸载向导并清空草稿；取消则保留当前步骤。
- 安装任务已经创建：无需确认，可切换到侧栏页面或服务器概览。任务继续由现有持久化 provisioning job 执行，并可由恢复通知重新进入。
- 创建成功：刷新服务器列表并返回服务器概览。

侧栏点击、页面返回按钮和其他会离开创建页面的入口都调用同一请求离开函数，避免行为不一致。

## 创建页面布局与语义

创建页面使用普通 `<section>` 和页面标题关联，不使用 `role="dialog"`。头部沿用现有视觉层级：

- 左侧：返回按钮、标题和说明。
- 中间：六步进度指示器。
- 右侧：返回服务器概览的关闭按钮。

市场详情模式可以隐藏重复标题区域，但必须保留明确的返回市场列表和退出创建入口。步骤指示器继续使用有序列表；仅已完成且允许返回的步骤可交互。

离开确认仍使用现有危险操作确认组件，因为它是短暂、阻塞且需要明确选择的操作，适合对话框语义。

## 向导状态契约

`CreateServerWizard` 增加最小状态通知，向 `AppShell` 表示 provisioning job 是否已创建。该通知只用于决定离开时是否确认，不把整个向导状态提升到父组件。

状态定义：

- `draft`：尚未创建 provisioning job，离开会丢失输入。
- `running`：job 已创建且未完成，允许离开并后台继续。
- `complete`：job 已完成，创建页面返回服务器概览。

现有进度、头部返回和市场详情通知继续保留，但名称从“Dialog Header”调整为与页面无关的“Page Header”，避免组件契约绑定模态框实现。

## BBSMC 来源接入

创建市场浏览器的来源类型扩展为 `Modrinth | BBSMC`。来源选择后，搜索、详情和版本请求通过小型 provider 分发函数调用对应现有 API：

- Modrinth：现有搜索、详情和版本接口。
- BBSMC：现有 `search_bbsmc_projects`、`get_bbsmc_project` 和 `list_bbsmc_versions`。

所有创建搜索都限定 `projectType: "modpack"`。加载器和排序筛选继续传递给提供商 API。

BBSMC 版本按文件能力分类：

- 至少一个文件提供允许的 `https://cdn.bbsmc.net/` 直链：版本可选择。
- 没有公共直链但提供 `diskUrls` 或标记为 `diskOnly`：版本禁用，显示“需要手动下载”。
- 既无直链也无外部链接：版本禁用，显示“没有可自动安装的文件”。

BBSMC API 当前不能可靠证明版本是专用服务端包，因此所有可选择的 BBSMC 版本按未验证包处理。用户必须在现有风险确认对话框中确认后才能进入下一步。

## BBSMC provisioning 规划

后端 `planMarketplacePack` 增加 BBSMC 分支：

1. 读取所选 BBSMC 版本。
2. 选择 primary 公共直链文件，否则选择第一个公共直链文件。
3. 使用现有 URL 信任边界验证协议和 BBSMC CDN 主机。
4. 建立 marketplace artifact，保存提供商、项目、版本、文件名、大小和哈希。
5. 添加需要确认的 `PACK_UNVERIFIED` 警告。
6. provisioning 下载阶段把文件保存到隔离的源目录，再交给现有 `planLocalPack` 分析和解压。

外部网盘 URL 永远不进入自动下载 artifact。即使前端状态被绕过，后端也必须拒绝没有允许直链的 BBSMC 版本。

## 错误处理

- BBSMC API 搜索、详情或版本请求失败时，在创建页面内显示原始操作错误，不伪造空结果。
- 公共直链缺失时显示不可安装原因，不允许产生选择事件。
- 后端 URL 或版本验证失败时中止规划，并在兼容性步骤前向用户显示错误。
- 离开确认失败不会清空草稿。
- 已开始的任务仍使用现有 provisioning 恢复通知和错误持久化。

## 无障碍与键盘行为

- 创建页面标题通过 `aria-labelledby` 与主内容关联。
- 返回、退出和步骤按钮均有本地化可访问名称。
- 移除创建流程的对话框焦点陷阱后，键盘用户可以正常访问侧栏；离开草稿时焦点进入确认对话框，取消后恢复到触发控件。
- 禁用的 BBSMC 版本提供可见原因和 `title`，并在版本文本中显示“需要手动下载”。
- 继续保持鼠标点击不显示焦点框、键盘导航显示 `:focus-visible`。

## 测试策略

前端组件测试：

- 点击创建按钮后，创建页面位于 `<main>` 中，不存在创建对话框或遮罩层。
- 六步进度和创建标题仍可访问。
- 草稿状态离开会打开确认；取消保留页面，确认返回概览。
- job 创建后侧栏导航不要求放弃确认。
- 文件拖入审查后进入内联创建页面。
- 来源下拉菜单包含 Modrinth 和 BBSMC，不包含无凭据 CurseForge。
- 切换 BBSMC 后调用 BBSMC 搜索、详情和版本命令。
- BBSMC 公共直链版本打开未验证确认，确认后产生 BBSMC 选择。
- BBSMC 网盘版本禁用并显示手动下载原因。

后端测试：

- BBSMC 公共 CDN 文件生成 marketplace provisioning plan。
- 规划包含 `PACK_UNVERIFIED` 且要求确认。
- 外部网盘版本和非允许域名被拒绝。
- artifact 下载后继续走现有本地整合包规划。

回归验证：

- TypeScript 检查。
- 完整 Vitest 测试。
- 生产构建。
- Electron UI 冷启动测试，检查创建页面位于主内容区、无渲染器错误、向导布局和键盘焦点正常。

## 风险与控制

- 风险：创建页面状态与侧栏导航分散。控制：所有离开入口使用同一请求离开函数。
- 风险：BBSMC 版本元数据不足。控制：默认未验证并要求明确确认，不推断服务端兼容性。
- 风险：外部下载链接绕过前端。控制：后端重新解析版本并独立验证 CDN 主机。
- 风险：安装中离开导致用户误以为任务停止。控制：保留现有恢复通知，并在创建任务后把运行状态持久化。
