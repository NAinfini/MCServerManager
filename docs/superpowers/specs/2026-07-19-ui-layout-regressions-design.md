# UI 布局回归修复设计

## 目标

修复预览一致性重构后出现的三处界面回归，同时保持现有业务逻辑、数据请求、EULA 流程和键盘可访问性不变。

## Java 运行时页面

- 宽屏采用两列主布局：托管 Java 占约三分之一，已安装运行时占约三分之二。
- 服务器兼容性和运行时扫描失败列表占据整行。
- 1100px 以下恢复单列，避免表格和中文文本被压缩。
- 通过语义类名表达区块角色，不再依赖 `nth-child`，防止条件渲染改变布局。

## 应用日志页面

- 筛选侧栏和日志列表容器始终渲染。
- 0 条记录时，将空状态放入列表面板内部。
- 没有选中日志时不渲染详情面板，并使用两列空状态布局；存在日志时恢复筛选、列表、详情三栏。
- 加载错误继续使用现有显式错误提示，不隐藏失败。

## Switch 与 Checkbox

- 通用按钮高度规则排除 `.switch-root` 与 `.checkbox-root`。
- Switch 保持 `32 × 18px`，Checkbox 保持 `16 × 16px`。
- 焦点样式仍仅使用 `:focus-visible`，保留键盘焦点，不增加鼠标点击焦点环。

## 测试与验收

- Java 样式测试验证角色类名和宽屏/窄屏网格。
- 日志组件测试验证空数据时筛选侧栏、列表面板和空状态仍存在，详情栏不存在。
- 控件样式测试验证通用按钮规则不会匹配 Switch/Checkbox。
- 运行相关测试、完整测试、生产构建和浏览器尺寸检查。

## 风险控制

- 仅修改三个界面的结构类名和对应样式。
- 不修改日志查询、Java 安装、设置持久化或桌面 IPC。
- 不增加依赖，不引入隐藏降级路径。
