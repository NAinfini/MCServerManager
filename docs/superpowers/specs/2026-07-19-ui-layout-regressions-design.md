# UI 布局回归修复设计

## 目标

修复预览一致性重构后出现的界面回归，同时保持现有业务逻辑、数据请求、EULA 流程和键盘可访问性不变。

## Java 运行时页面

- 宽屏采用两列主布局：托管 Java 占约三分之一，已安装运行时占约三分之二。
- 服务器兼容性和运行时扫描失败列表占据整行。
- 1100px 以下恢复单列，避免表格和中文文本被压缩。
- 通过语义类名表达区块角色，不再依赖 `nth-child`，防止条件渲染改变布局。

## 应用日志页面

- 筛选侧栏和日志列表容器始终渲染。
- 0 条记录时，将空状态放入列表面板内部。
- 没有选中日志时不渲染详情面板，并使用两列空状态布局；存在日志时恢复筛选、列表、详情三栏。
- 加载错误继续使用现有显式错误提示，不隐藏失败。

## Switch 与 Checkbox

- 通用按钮高度规则排除 `.switch-root` 与 `.checkbox-root`。
- Switch 保持 `32 × 18px`，Checkbox 保持 `16 × 16px`。
- 焦点样式仍仅使用 `:focus-visible`，保留键盘焦点，不增加鼠标点击焦点环。

## 市场卡片与详情截图

- 降低预览层通用按钮高度规则的选择器优先级，使市场卡片自己的最小高度能够生效。
- 市场根容器使用纵向 Flex 布局：筛选栏和任意数量的错误提示保持内容高度，结果区占满剩余空间，避免提示与首行卡片重叠。
- 市场结果区使用内容高度的隐式网格行并独立滚动，避免把全部卡片均分进固定高度。
- 宽屏显示三列卡片，中等宽度显示两列，窄屏显示一列。
- 卡片使用真实内边距和网格间距；封面不再通过外边距溢出固定网格列，确保封面、标题和操作按钮之间保留一致间距。
- 卡片封面保持方形并使用 `object-fit: contain`，不裁切、不拉伸。
- 详情截图使用 `object-fit: contain`，保留双列画廊和原图比例；允许留白，不再裁切横图、方图或超宽图。

## 全应用文字与控件间距

- 900px 以下将应用壳、运行状态栏、侧栏和主页面全部重置到单列，防止固定网格位置重新生成隐式第二列。
- 480px 以下隐藏六个向导步骤标签，仅保留步骤圆点，避免中英文标签挤出标题栏。
- 底部状态栏使用可收缩三列；状态文案有明确的 `min-width: 0` 和省略策略，图标不收缩。
- 900px 以下控制台侧栏移动到终端下方，计划任务操作区移动到独立行并允许换行。
- 侧栏折叠按钮显式保持 `30 × 30px` / `32 × 32px`，不受通用按钮最小高度覆盖。
- 向导和设置错误使用统一块级反馈间距并允许长错误文本断行；嵌入设置页的子面板移除重复外框和阴影。
- 不批量替换装饰性像素值，也不移除列表名称的合理省略号；仅处理有布局证据的裁切和挤压。

## 测试与验收

- Java 样式测试验证角色类名和宽屏/窄屏网格。
- 日志组件测试验证空数据时筛选侧栏、列表面板和空状态仍存在，详情栏不存在。
- 控件样式测试验证通用按钮规则不会匹配 Switch/Checkbox。
- 市场样式测试验证动态提示的 Flex 高度链、三列/两列/单列断点、内容高度网格行、卡片间距，以及卡片和详情图片的 `contain` 行为。
- 响应式样式测试验证应用壳单列位置、向导标签、状态栏收缩、控制台和任务操作区、侧栏按钮尺寸及反馈块边缘。
- 运行相关测试、完整测试、生产构建和浏览器尺寸检查。

## 风险控制

- 仅修改相关界面的结构类名和对应样式。
- 不修改日志查询、Java 安装、设置持久化或桌面 IPC。
- 不增加依赖，不引入隐藏降级路径。
