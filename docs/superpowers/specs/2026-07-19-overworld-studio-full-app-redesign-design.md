# Overworld Studio 全应用重设计

## 目标

- 将现有桌面应用统一为“清晰专业，同时保留 Minecraft 辨识度”的 Overworld Studio 视觉系统。
- 保留并强化完整开服路径：选择或拖入服务端包、自动识别版本、安装服务端文件、用户确认 EULA、配置资源、创建并启动服务器。
- 解决已观察到的文字裁切、底部空白、嵌套滚动、按钮比例失衡、长日志挤压和版本列表重叠问题。
- 提供浅色、深色与跟随系统三种主题；功能、信息层级和状态含义在两种主题中保持一致。

## 已确认的视觉方向

采用 B“Overworld Studio”平衡型方案：专业桌面工具结构，使用低饱和苔藓绿、石板灰和少量金色强调 Minecraft 氛围。识别度来自方块几何、地图工作台式层级、状态符号和克制的地形图形，不使用像素字体、大面积游戏贴图或高饱和霓虹。

设计原则：

- 稠密但不拥挤：控制台、日志和版本列表使用紧凑信息密度；创建向导和设置保留更宽松的操作空间。
- 平面优先：依靠边框、表面色和排版建立层级，阴影只用于窗口、浮层和危险确认。
- 状态明确：运行、停止、警告和失败同时使用图标、文字与颜色。
- 键盘优先：只使用 `:focus-visible` 显示焦点环，鼠标点击不保留相同焦点视觉。

## 信息架构

全局侧栏只保留：

1. 服务器
2. Java 运行时
3. 应用日志
4. 设置

市场不是全局页面。它只在两个上下文中出现：

- 创建模组包服务器：`来源 → 浏览市场`
- 编辑现有模组包服务器：`服务器详情 → 内容 → 浏览市场`

继续复用 `AppShell` 的 `activePage`、`selectedServerId` 和现有 Zustand 服务器标签状态，不为本次重设计引入路由库。设置页增加独立的二级菜单状态，分类为常规、外观、文件与下载、创建与安装、更新与诊断、维护操作。

## 应用外壳与仪表盘

- 保留原生窗口标题栏、全局侧栏、运行时状态栏和底部状态栏。
- 服务器仪表盘展示运行状态摘要、服务器卡片或表格、运行时健康和最近活动。
- 服务器身份、状态和主要操作使用稳定位置；列表和卡片模式保留现有切换能力。
- 可滚动区域必须由明确的 `min-height: 0` 容器承载，应用根节点不能通过固定 `min-height` 与 Electron 最小窗口高度相互冲突。

## 服务器详情工作区

保留六个现有标签：控制台、文件、内容、备份、设置、活动。服务器身份、Minecraft/加载器版本、端口、内存和启动操作固定在工作区顶部。

控制台采用主日志区与健康侧栏：

- 主区展示实时日志、搜索、复制、清理视图和命令输入。
- 侧栏展示 CPU、内存、TPS、延迟、在线玩家与安全快捷操作。
- 低宽度下侧栏移动到日志下方，不形成横向滚动。

命令自动补全在现有 `MC_COMMANDS`、命令历史和 `CommandSuggestions` 基础上扩展：

- 输入 `/` 或命令片段后显示命令、说明和参数格式。
- 支持方向键选择、Tab 接受、Enter 执行、Escape 关闭和历史命令浏览。
- 第一阶段必须把来源标为“内置命令目录”，并按可用的 Minecraft/加载器元数据过滤；现有后端只向进程 stdin 写命令，没有实时查询服务端命令树的能力，因此不得声称建议来自当前服务端。
- 只有新增可验证的后端命令发现协议后，才能显示“当前服务端命令”。

## 创建服务器向导

六个步骤固定在主内容区头部：来源、兼容性、Java、服务器配置、检查与 EULA、安装并启动。

来源支持：

- 拖入或选择服务端整合包
- 新建空白服务器
- 导入已有文件夹
- 打开本地整合包文件
- 在上下文市场浏览服务端整合包

加载器从首版支持 Vanilla、Paper、Fabric、Forge、NeoForge 与 Quilt。应用继续自动识别 Minecraft、加载器和 Java 版本，但识别结果必须展示来源和不确定性。

若无法确认是专用服务端包，显示明确风险、原因和未勾选的用户确认框；用户确认后可以继续。BBSMC 公共直链版本可选择，外部网盘版本禁用直接选择并显示手动下载说明。CurseForge 没有 API 密钥时保持不可用并说明原因。

向导尚未安装时离开，显示“放弃创建”确认。安装任务开始后允许离开，任务继续在后台运行并可恢复。第一步不显示重复的底部“上一步”；后续步骤保留返回能力。

EULA 必须由用户主动打开并勾选确认；确认框不默认选中。只有明确确认后才允许写入 `eula=true` 和启动服务器。

## 上下文市场与整合包详情

市场打开后直接显示搜索、来源、加载器、Minecraft 版本和排序筛选，不显示编辑推荐横幅。

结果使用双列大卡片：大封面、标题、作者、简介、来源、下载量、Minecraft 版本、加载器、服务端可用性和明确操作按钮。BBSMC 没有可靠关注数据时不显示“0 关注”。公共直链显示可安装，外部网盘显示“需要手动下载”。

详情保持在主内容区：

- 顶部展示项目身份、来源、作者、兼容性和主要版本操作。
- 媒体画廊使用固定比例与 `object-fit: cover`，不产生图片内部滚动条。
- Markdown 延续现有安全清理与图片代理；加载失败显示明确占位和原始链接。
- 正文限制阅读宽度，远程标题不能超过页面主标题层级。
- 右侧服务端版本栏按版本名、发布名称、Minecraft、加载器、文件状态和不可用原因分层，窄屏移到正文之前。

## Java 运行时

Java 运行时作为完整主内容页展示：已发现运行时、兼容性覆盖、服务器占用情况、路径和来源。

- 托管 Temurin 安装需要用户明确勾选同意，安装到应用数据目录，不修改系统 PATH 或注册表。
- 保留 Oracle Java 官方下载入口，安装后可重新扫描本机 Java。
- 长路径截断但可查看完整值，不挤压操作按钮。
- 创建向导给出推荐 Java，但启动前展示最终分配结果。

## 设置

设置使用主内容区内的左侧二级菜单，不使用顶部标签或长单页。外观分类提供浅色、深色和跟随系统。

主题通过根节点数据属性与 CSS 变量实现，至少覆盖：背景、表面、边框、主文本、次文本、强调色、成功、警告、错误、焦点环、阴影。主题选择持久化到现有设置存储，跟随系统时监听系统主题变化。

所有设置即时保存并显示明确状态。路径控件允许换行或省略并提供完整值；危险维护操作与普通设置分开，执行前展示影响范围。

## 应用日志

应用日志采用三级布局：左侧筛选、中间紧凑事件列表、右侧详情。

- 长消息在列表中单行省略，完整文本只在详情显示。
- 级别标签为固定宽度的独立元素，不参与正文压缩。
- 相同消息、来源和堆栈签名的重复事件分组并显示次数；不同堆栈不能被合并。
- 日志详情提供上下文、堆栈、复制和导出操作。
- 清除日志继续使用危险确认；错误不静默隐藏。

## 通用状态与安全交互

- 加载态使用稳定骨架或进度，不改变布局高度。
- 空状态说明原因并提供一个主要下一步。
- 失败态展示根因、重试与日志入口，不提供假成功或静默回退。
- 后台安装显示阶段、百分比、目标目录和任务入口。
- 确认对话框只用于放弃未保存流程、清理数据、停止服务器和 EULA 等真正需要决定的节点。
- 对话框必须管理焦点、支持 Escape、返回触发元素并提供明确按钮顺序。

## 响应式与裁切规则

- 目标覆盖 Electron 最小窗口以及 1280×720、1440×900、1920×1080。
- 所有 Grid/Flex 主内容子项默认 `min-width: 0`；垂直滚动容器默认 `min-height: 0`。
- 页面只保留一个主要纵向滚动区；控制台、日志详情等专业工具区域可拥有受控内部滚动。
- 标题、路径、版本名、远程 Markdown 和错误文本必须显式换行、省略或 `overflow-wrap:anywhere`。
- 不通过固定大高度填充页面，不保留无内容的底部操作区。
- 低宽度下双列市场卡片改为单列，详情侧栏移动到主内容下方，设置二级菜单可收窄但不移回顶部。

## 组件与代码边界

- 继续复用现有 Button、Dialog、Radix 控件、TanStack Query、Zustand 和 i18n，不新增 UI 框架。
- 在 `src/styles.css` 建立主题变量、布局尺寸和状态令牌；组件使用语义类名，不复制页面级颜色常量。
- AppShell 只负责全局导航和页面选择；服务器详情、创建向导、设置二级菜单和市场上下文各自管理局部状态。
- 市场搜索和详情继续复用现有 `marketplaceApi`、`MarketplaceMarkdown`、provider branding 和图片代理。
- 后端安装、数据库、EULA、进程管理和服务端文件操作保持现有契约；本次重设计不以视觉变化为理由改写核心 provisioning 流程。

## 数据流与错误边界

页面继续通过 TanStack Query 调用 Electron preload 命令。查询错误在当前工作区内展示，并提供重试；命令或安装 mutation 保留原错误对象和日志关联标识。后台任务状态由现有创建生命周期与任务数据驱动，页面离开不能取消任务或伪造完成。

主题和纯界面偏好由设置存储管理；服务器选择、详情标签和局部筛选继续由组件或 Zustand 管理。远程内容必须经过现有清理和代理边界，外部链接只允许受支持协议并由 Electron 打开。

- 增加布局契约测试：`min-width:0`、`min-height:0`、主滚动区、市场双列/单列、设置左侧菜单、版本列表与长日志。

- 运行相关 Vitest、Electron 后端测试、Electron UI smoke、生产构建和 `git diff --check`。

## 实施顺序

1. 主题令牌、全局外壳与滚动约束。
2. 仪表盘与服务器详情工作区。
3. 控制台与命令补全交互。
4. 创建向导和上下文市场。
5. 整合包详情与媒体/版本布局。
6. 设置左侧菜单、Java 运行时和应用日志。
7. 通用状态、安全对话框、响应式与完整回归测试。

## 非目标

- 不新增 CurseForge API 密钥或绕过其接口要求。
- 不伪造 BBSMC 关注数、版本可安装性、服务端包验证或命令来源。
- 不替换 Electron、React、现有数据库或安装后端。
- 不在本轮加入账户、云同步、远程主机管理、权限系统或多人协作。
