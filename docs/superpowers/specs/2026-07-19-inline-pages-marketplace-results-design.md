# 主内容页与市场结果修复设计

## 目标

- Java 运行时和设置从全屏弹窗迁移到应用主内容区，保留侧栏、顶栏和状态栏。
- 创建服务器的市场浏览区占满可用高度，并移除市场首页底部重复的“上一步”。
- BBSMC 空搜索展示公开整合包列表，而不是暗中搜索固定关键词 `server`。

## 设计

`AppShell` 继续使用现有 `activePage` 作为唯一页面状态。选择 Java 或设置时直接切换 `activePage`，主内容区分别渲染 `JavaRuntimesView` 和 `SettingsView`；删除两套弹窗状态、弹窗 JSX 及其专用样式。创建服务器仍优先于普通页面，因此离开草稿时继续复用现有放弃确认逻辑。

市场浏览器外层补上已有的 `wizard-marketplace-step` 容器，使现有的 `height: 100%` 和 `max-height: none` 规则真正生效。市场首页不渲染底部导航栏，因为顶部已有返回入口；后续步骤的上一/下一步行为不变。

BBSMC 的默认发现查询改为空字符串。前端允许 BBSMC 在空查询时执行查询，后端也将空查询传给公开 API；用户主动提交空白搜索仍保持按钮禁用。Modrinth 的默认发现逻辑不变。

## 验证

- 组件测试确认 Java/设置位于 `main` 内且不存在 `dialog`。
- 向导测试确认市场容器存在、底部“上一步”不存在、顶部返回回调可用。
- 前后端测试确认 BBSMC 默认请求使用空查询并保留模组包筛选。
- 运行相关测试、类型检查和构建。
