# 市场项目详情页重设计

## 目标

- 将模组与整合包详情页从基础信息堆叠升级为具有清晰视觉层级的编辑式混合布局。
- 同时服务“了解项目”和“选择可安装服务端版本”两个任务，不改变现有创建服务器流程。
- 保留 Modrinth 与 BBSMC 数据、图片代理、安全清理、版本可安装性判断及用户确认行为。

## 视觉方向

采用已确认的“编辑式混合布局”。继续使用应用现有冷色深灰和绿色强调色，不引入新字体、图片服务、动画库或设计依赖。视觉深度来自项目图标的模糊环境背景、分层表面、非对称截图布局和更明确的排版比例，而不是渐变按钮、强阴影或大面积高饱和色。

## 页面结构

详情页仅保留一个主内容滚动区，右侧版本栏独立滚动。顶部返回按钮位于内容容器外沿，下面依次为项目头部、统计带和内容/版本双栏。

项目头部使用语义化 `header`。项目图标同时作为低透明度、模糊的环境背景，前景显示 88–96 px 图标、来源标签、24 px 左右的项目标题、两行内简介和外部项目页链接。背景必须有遮罩，不能影响文字对比度；图标失败时继续显示现有字母占位。

统计带使用四个等宽或自适应单元，分别显示下载量、关注数、模组数量（有数据时）和 Minecraft 版本。数字使用等宽数字特性，标签与数值分层显示；缺失数据不伪造，模组数量未知时隐藏该单元。

主体在宽窗口中使用 `minmax(0, 1fr) / 300–340 px` 双栏。左侧按“截图”和“项目介绍”组织；右侧版本栏在详情容器内保持可见并独立滚动。窄窗口下改为单栏，版本区域放在项目介绍之前，避免用户滚过长正文后才能安装。

## 截图与正文

截图区域使用语义化 `section` 和可见标题。第一张截图占较大网格单元，其余截图使用较小单元；所有单元限制高度并保持固定比例，不生成横向滚动条。只有一张截图时宽度仍受限，不铺满整个窗口。图片继续复用现有 BBSMC 后端代理与失败处理。

正文放入独立的介绍区域，文本最大阅读宽度约 72–76 字符。标题按视觉层级缩放，但不让远端 Markdown 的 `h1` 超过页面项目标题；段落、列表、引用、代码和链接使用稳定的垂直节奏。`details` 元素显示为可辨识的折叠行，保留刷新后展开状态。正文图片最大高度继续受限，不产生嵌套滚动。

## 版本栏

版本栏使用语义化 `aside`，头部显示“版本”和总数。每个版本按钮分成版本号、发布名称、Minecraft 版本、服务端包状态和不可用原因五层；下载图标保持顶部对齐。可安装项提供清晰的 hover、active 与 `focus-visible` 状态，禁用项保留原因文本和 `title`，不伪装成可点击状态。

## 响应式与无障碍

- 保留顶部返回按钮和外部链接的可访问名称。
- 有意义的项目图标和截图提供可用替代文本；纯装饰背景使用 `aria-hidden`。
- 只使用 `:focus-visible`，鼠标点击不显示焦点框，键盘操作始终可见。
- 所有长标题、简介、版本名和用户提供的 Markdown 必须换行或截断，不能造成水平滚动。
- 继续遵守 `prefers-reduced-motion`；新增过渡仅使用 `transform`、`opacity`、边框色或背景色，不使用 `transition: all`。
- 低于约 1100 px 的可用宽度时切换单栏；截图网格和统计带继续自适应。

## 数据与错误处理

不新增后端字段或远程请求。组件继续消费 `ProjectDetails`、`ProjectVersion[]` 和现有 provider 值。加载态继续使用现有 `LoadingState`，查询错误继续显示当前内联错误。没有截图或正文时不渲染空标题/空面板；没有版本时显示现有不可安装提示。

## 验证

- 组件测试确认语义化项目头部、统计带、截图区、介绍区和版本栏存在，并保留返回、外部链接和版本选择行为。
- 样式契约测试确认双栏、窄屏单栏、受限截图、正文阅读宽度、`focus-visible` 和无横向画廊滚动。
- Markdown 测试确认标题、折叠区域、安全清理与 BBSMC 图片代理行为不回归。
- 运行相关 Vitest 测试、Electron 后端测试、生产构建和 `git diff --check`。

## 非目标

- 不新增收藏、评论、作者资料、标签页、灯箱、轮播、下载缓存或版本搜索。
- 不更改 Modrinth/BBSMC 搜索、安装协议、EULA、服务端包警告和后台任务流程。
- 不重构整个创建服务器向导，也不更换全局主题或字体系统。
