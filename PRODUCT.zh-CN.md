# 产品

[English](PRODUCT.md)

## 注册

product

## 用户

从本地桌面机器运行 Minecraft 服务器的人。他们需要创建配置档、启动和停止服务器、查看日志、管理文件、检查 Java 兼容性、维护备份，而不必在终端命令和文件夹之间来回切换。

## 产品目的

MC Server Manager 是本地 Minecraft 服务器运维的桌面控制台。成功标准是用户能一眼看到服务器健康状态，快速完成常见操作，并在失败时理解原因，而不是翻找原始运行状态。

## 品牌个性

稳重、运维感、技术向。

## 反向参考

避免营销页视觉、超大 hero 区块、装饰性渐变、玩具化 Minecraft 模仿，以及让日常操作需要长滚动的低密度表单。

## 设计原则

- 配置细节之前先展示状态。
- 使用熟悉的桌面工具模式，而不是装饰性新奇交互。
- 重复服务器操作使用紧凑、对齐的控件。
- 错误必须明确且可恢复。
- 在界面中解释新手设置步骤：Java、`server.jar`、EULA、启动、备份。
- 所有控件保留键盘和屏幕阅读器访问能力，但不使用浏览器默认 focus outline。
- 破坏性或中断性操作必须明确确认。

## 可访问性与包容性

目标是 WCAG AA 对比度、减少动效和不依赖颜色单独表达状态。键盘导航应保持可用，但默认 focus outline 会被刻意抑制，以维持应用的桌面视觉风格。

## 更新与内容策略

- 应用更新从 GitHub Releases 检查，并且只在用户确认后安装。
- 托管服务器运行时阻止安装应用更新。
- mod、插件、整合包和服务端 jar 不会被静默自动安装。
- 用户自行选择可信的服务端 jar 下载来源；应用可以把用户提供的 jar 安装为 `server.jar`。
- 用户必须自行阅读并接受 Minecraft EULA，之后才应把 `eula` 设置为 `true`。
- 已安装内容更新是手动流程：先检测更新，再全部更新或单项更新。
