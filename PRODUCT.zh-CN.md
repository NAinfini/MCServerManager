# 产品

[English](PRODUCT.md)

## 注册

product

## 用户

从本地桌面机器运行 Minecraft 服务器的人。他们需要创建配置档、启动和停止服务器、查看日志、管理文件、检查 Java 兼容性并维护备份，而不必在终端命令和文件夹之间来回切换。

## 产品目的

MC Server Manager 是本地 Minecraft 服务器运维的桌面控制台。成功标准是用户能一眼看到服务器健康状态，快速完成常见操作，并在失败时理解原因，而不是翻找原始运行状态。

## 品牌个性

稳重、运维感、技术向。

## 反向参考

避免营销页视觉、超大 hero 区块、装饰性渐变、玩具化 Minecraft 模仿，以及让日常操作需要长滚动的低密度表单。

## 设计原则

- 配置细节之前先展示状态。
- 使用熟悉的桌面工具模式，而不是装饰性新奇交互。
- 重复服务器操作使用紧凑、对齐的控件。
- 错误必须明确且可恢复。
- 用一条可恢复流程解释新手设置：来源、兼容性、Java、配置、明确确认 EULA、安装和启动就绪。
- 所有控件保留键盘和屏幕阅读器访问能力，但不使用浏览器默认 focus outline。
- 破坏性或中断性操作必须明确确认。

## 可访问性与包容性

目标是 WCAG AA 对比度、减少动效和不依赖颜色单独表达状态。键盘导航应保持可用，但默认 focus outline 会被刻意抑制，以维持应用的桌面视觉风格。

## 发布策略

- tag release 通过 GitHub Actions 发布 Windows、Linux 和 macOS 产物。
- macOS CI 产物在配置签名凭据前默认未签名。
- 发布元数据必须与自动更新行为和已发布产物保持一致。

## 更新与内容策略

- 应用更新从 GitHub Releases 检查，并且只在用户确认后安装。
- 托管服务器运行时阻止安装应用更新。
- 只有用户通过 Modrinth 和 CurseForge 或本地入口选择、拖入、导入、发现或配置来源后，服务端配置流程才会开始。
- Vanilla、Paper、Forge、NeoForge、Fabric 和 Quilt 使用明确的加载器适配器，并限制下载主机且验证安装输出。
- 优先使用专用服务端包；未验证压缩包必须显示服务端包警告并由用户确认。
- 安装托管 Eclipse Temurin 需要单独同意，且不会修改系统 `PATH`。
- 用户必须阅读 Minecraft 条款并明确确认 EULA；应用不会推断、预选或代替接受。
- 解压后会先删除整合包自带脚本，再安装加载器；应用绝不执行这些脚本。
- 已安装内容更新是手动流程：先检测更新，再全部更新或单项更新。
